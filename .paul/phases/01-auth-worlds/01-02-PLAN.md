<objective>
## Goal
Implement the full authentication flow: sign up, sign in, sign out, and route protection via middleware. Users can create an account and access a protected app shell.

## Purpose
Auth is the gate to everything — worlds, entities, and collaboration all require knowing who the user is. This plan establishes identity so 01-03 can build world CRUD against authenticated users.

## Output
- Sign up page with name/email/password form
- Sign in page with email/password form
- Middleware protecting `/app/*` routes
- Protected app shell with sign-out
- Redirect unauthenticated users to sign-in
</objective>

## AC-1: User can sign up
```gherkin
Given a visitor on /sign-up
When they enter name, email, password and submit

```

## AC-2: User can sign in
```gherkin
Given a registered user on /sign-in
When they enter valid email and password
Then they are authenticated and redirected to /app
```

## AC-3: Invalid credentials show error
```gherkin
Given a visitor on /sign-in
When they enter invalid credentials
Then an error message is displayed and they remain on /sign-in
```

## AC-4: Unauthenticated users are redirected
```gherkin
Given a visitor who is not signed in
When they navigate to /app or any /app/* route
Then they are redirected to /sign-in
```

## AC-5: User can sign out
```gherkin
Given an authenticated user in the app shell
When they click the sign out button
Then their session ends and they are redirected to /sign-in
```
