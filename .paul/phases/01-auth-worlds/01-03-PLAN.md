<task type="auto">
  <name>Task 1: Create Convex mutations and queries for worlds and memberships</name>
  <files>convex/worlds.ts, convex/memberships.ts</files>
  <action>
    `convex/worlds.ts`:
    - `list` query: return all worlds where user is owner OR has a membership. Join with memberships table to get user's role for each world.
    - `get` query: return a single world by ID. Verify the requesting user is a member (owner or membership exists). Return null or throw if not a member.
    - `create` mutation: accept { name, description?, visibility }. Create world with ownerId = current user. Also create a membership record with role "owner", invitedAt = now, joinedAt = now. Return the new world ID.

    `convex/memberships.ts`:
    - `listByWorld` query: return all memberships for a world ID, with user info (name, email) joined. Verify requesting user is a member of the world.
    - `invite` mutation: accept { worldId, email, role }. Verify requesting user is the world owner. Look up user by email. If found, create membership with joinedAt = null. If not found, still create membership with userId pointing to... actually, we need the user to exist. Create the membership only if user exists. Return { success: true, found: true } or { success: true, found: false } with a message "User not found — they'll need to sign up first".

    Use `auth.getUserId(ctx)` for all auth checks. Throw ConvexError for unauthorized access.

    Avoid: complex invitation token system — MVP just creates a membership record for existing users.
    Avoid: allowing self-invite or duplicate memberships.
  </action>
  <verify>`npx tsc --noEmit` passes; convex dev shows no errors</verify>
  <done>AC-1 (backend), AC-4 (backend), AC-5 (backend) satisfied</done>
</task>
