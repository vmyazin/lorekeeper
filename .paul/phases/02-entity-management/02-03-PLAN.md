<task type="auto">
  <name>Task 1: Create EditEntityDialog and DeleteEntityDialog components</name>
  <files>components/entities/EditEntityDialog.tsx, components/entities/DeleteEntityDialog.tsx</files>
  <action>
    **EditEntityDialog (components/entities/EditEntityDialog.tsx):**
    Follow CreateEntityDialog pattern (Radix Dialog, FormData, useMutation).
    - Props: entity (full entity object with all fields), entityType (string), entityLabel (string)
    - Trigger: Button with Pencil icon, "Edit"
    - Form fields:
      - name (Input, required, pre-filled)
      - description (textarea, pre-filled)
      - Type-specific fields based on entityType:
        - character: title, appearance, personality, background (all textareas)
        - place: geography, climate, significance (all textareas)
        - faction: goals, beliefs, headquarters (all textareas)
        - artifact: origin, powers, currentLocation (all textareas)
        - lore: category (Input), source (Input)
      - Use a config map: `entityFields[entityType]` returns array of { name, label, type: "input" | "textarea" }
    - On submit: call api.entities.update({ id: entity._id, entityType, ...changedFields })
    - Close dialog on success

    **DeleteEntityDialog (components/entities/DeleteEntityDialog.tsx):**
    Simple confirmation dialog.
    - Props: entityId (string), entityType (string), entityName (string), worldId (string), onDeleted callback
    - Trigger: Button with Trash2 icon, variant="destructive", "Delete"
    - Dialog body: "Are you sure you want to delete [name]? This action cannot be undone."
    - Confirm button: "Delete [entityLabel]" with destructive variant
    - On confirm: call api.entities.remove({ id: entityId, entityType })
    - On success: call onDeleted() (parent navigates back to list)
  </action>
  <verify>TypeScript compiles: `npx tsc --noEmit`</verify>
  <done>Components created, ready for use in detail page. Supports AC-2, AC-3.</done>
</task>
