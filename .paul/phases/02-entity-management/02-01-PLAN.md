<task type="auto">
  <name>Task 1: Add entity tables to Convex schema and create unified CRUD functions</name>
  <files>convex/schema.ts, convex/entities.ts</files>
  <action>
    **Schema (convex/schema.ts):**
    Add 5 entity tables — characters, places, factions, artifacts, lore — each with:
    - worldId: v.id("worlds")
    - name: v.string()
    - description: v.optional(v.string()) — rich text stored as string
    - entityType: v.literal("character"|"place"|"faction"|"artifact"|"lore") — matches table name
    - createdBy: v.id("users")
    - createdAt: v.number()
    - updatedAt: v.number()
    - Index: by_world on ["worldId"]

    Entity-specific optional fields:
    - characters: title (role/title), appearance, personality, background
    - places: geography, climate, significance
    - factions: goals, beliefs, headquarters
    - artifacts: origin, powers, currentLocation
    - lore: category, source

    All extra fields are v.optional(v.string()).

    **CRUD (convex/entities.ts):**
    Create a single unified file with generic entity functions. Use a helper to validate entity type and resolve table name.

    - `list({ worldId, entityType })` — query: check membership, filter by worldId using index, filter by entityType at table level (each table IS a type). Actually simpler: query the specific table by worldId index.
    - `get({ id, entityType })` — query: get entity, check membership via worldId
    - `create({ worldId, entityType, name, description, ...typeSpecificFields })` — mutation: check membership + role (editor/owner), insert into correct table
    - `update({ id, entityType, ...fields })` — mutation: get entity, check membership + role, patch
    - `remove({ id, entityType })` — mutation: get entity, check membership + role, delete

    Use auth.getUserId + membership index for all permission checks. Viewers can read (list, get), editors/owners can write (create, update, remove).

    Avoid: Don't use a single generic "entities" table — keep separate tables per type for clean indexing and type safety.
  </action>
